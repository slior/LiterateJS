This page shows a simple example of handling an error. A simple code block could be something like: 


var x = 2 + 1
But it can also contain errors, for example: 


var y = x() //should result in a type error
These errors should be reported by LitJS, in whatever is configured.
By default (and currently the only existing way) - raise an alert in the browser.
So this last code section should have raised an alert window.
Similarly, a block that raises an exception explicitly, should have this exception shown as any other error (default - an alert window): 


throw "This is an error"
Note also that the error message includes an id of the block the error occurred in.
This id can be given by the user, or auto-generated by LitJS
This is an example of an error thrown from a block with an id given by the user ("MyCodeBlock"): 

throw "this is my error"
