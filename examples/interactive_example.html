Interactive Literate Programming

One can include a code block that depends on an input: 


var x = 5 * y;

which depends on the following input (y):

And the value of x updates when value of y changes: x.
In order for an input to be accessible from LitJS code blocks, it needs to have the lit-value class assigned to it.
It should also have an id that is a valid Javascript identifier, and unique in this page. This id will be used to refer to the input in other code blocks.
In the above example, the input is defined as: 

<input type=number id="y" value="5" class="lit-value"/>
This should also take care of responding to inputs changing. Inline code blocks should be updated automatically.

As another example, let's try the fibonacci example, but driven by an input. The simple (non-tail recursive, no memoization) fibonacci series implementation is given in this code: 


function fib(n)
{
	if (n < 0) return 0
	if (n == 0)  return 1
	if (n == 1)  return 1
	return fib(n-1) + fib(n-2)
}

So given the value of num:
We can calculate the numth fibonnacci number to be: fib(num-2) + fib(num-1) = fib(num)
Note: this is a very inefficient implementation, so even medium sized inputs will perform poorly.
